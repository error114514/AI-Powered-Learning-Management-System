# 智能化学习管理系统 — 课程管理模块“查看考勤”功能设计方案

## 1. 功能需求描述

在“课程管理”模块的每个课程后面增加一个按钮：“查看考勤”。  
点击该按钮后，页面显示该课程内所有用户的考勤详情，数据包括：

| 字段       | 说明                   |
|------------|------------------------|
| 考勤ID     | 唯一标识某次考勤记录   |
| 课程名     | 该条考勤所属的课程     |
| 用户名     | 参与考勤的学员用户名   |
| 考勤时间   | 该用户考勤的时间       |

---

## 2. 数据库设计

### 2.1 新增表结构：`attendance_records`

| 字段名        | 类型           | 说明                     | 备注                |
|---------------|----------------|--------------------------|---------------------|
| `id`          | bigint(20)     | 主键，自增               | 唯一考勤记录ID      |
| `course_id`   | bigint(20)     | 关联课程ID               | 外键，指向课程表    |
| `user_id`     | bigint(20)     | 参与考勤用户ID           | 外键，指向用户表    |
| `attendance_time` | datetime     | 考勤时间                 | 记录用户考勤时间    |
| `created_at`  | timestamp      | 记录创建时间             | 默认当前时间        |
| `updated_at`  | timestamp      | 记录更新时间             | 默认当前时间        |

---

## 3. 主要开发点

### 3.1 后端接口

- **接口1：新增考勤记录**

  - 路径：`POST /api/attendance`
  - 请求体：

    ```json
    {
      "course_id": 123,
      "user_id": 456,
      "attendance_time": "2024-04-01T08:00:00"
    }
    ```

  - 功能：保存用户的考勤记录到表 `attendance_records`

- **接口2：获取指定课程考勤列表**

  - 路径：`GET /api/courses/{course_id}/attendance`
  - 返回字段：

    ```json
    [
      {
        "id": 1,
        "course_name": "课程A",
        "user_name": "张三",
        "attendance_time": "2024-04-01 08:00:00"
      },
      ...
    ]
    ```

  - 逻辑：查询 `attendance_records` 按 `course_id`，关联课程表和用户表，返回对应数据

---

### 3.2 前端页面

- 在**课程管理模块**的课程列表每行末尾新增一个按钮 `查看考勤`  
- 点击按钮，调用接口 `GET /api/courses/{course_id}/attendance`  
- 展示一个模态框或跳转到新页面，展示考勤列表，字段包括：

  - 考勤ID
  - 课程名
  - 用户名（支持点击可跳转用户详情）
  - 考勤时间

- 风格保持与系统统一，使用系统现有的 UI 组件库（如 Element UI、Ant Design 等）

---

### 3.3 考勤录入功能扩展

- 系统已有的考勤成功逻辑中，调用接口新增考勤记录，保证数据同步写入数据库

---

## 4. 设计说明及注意事项

- 每个课程独立查看，只拉取该课程对应的考勤数据  
- 考勤数据按时间降序排列，方便查看最新记录  
- 需做好权限控制，确保只有有权限查看该课程的用户能查看考勤  
- 保持接口统一的返回格式，方便前端统一处理  
- 建议数据库表加索引优化查询效率，特别是 `course_id` 和 `user_id` 列  

---

## 5. 示例 SQL 建表语句

```sql
CREATE TABLE attendance_records (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  course_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  attendance_time DATETIME NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_course_id (course_id),
  INDEX idx_user_id (user_id),
  FOREIGN KEY (course_id) REFERENCES courses(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
